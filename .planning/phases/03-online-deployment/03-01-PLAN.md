name: Build Check

on:
  pull_request:
    branches: [main]

  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v6
      - uses: actions/setup-python@v6
        with:

      - run: pip install -r requirements.txt
      - run: mkdocs build --strict

- Use `cache: 'pip'` for faster CI runs
- Unlike Netlify, GitHub Actions does NOT auto-install from requirements.txt -- explicit `pip install -r requirements.txt` step IS required here
- Use `mkdocs build --strict` (NOT plain `mkdocs build`) -- `--strict` treats warnings as errors for stricter CI merge gating

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Public internet -> Netlify CDN | Untrusted browsers access the site; responses must include security headers |
| Netlify build environment | Trusted build system with access to repo contents and deploy credentials |
| GitHub Actions runner | Trusted CI environment; runs build verification only, no deployment access |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Spoofing | HTTP access | mitigate | HSTS header (max-age=31536000; includeSubDomains; preload) + Netlify auto-HTTPS via Let's Encrypt (D-12) |
| T-03-02 | Tampering | Site responses (clickjacking) | mitigate | X-Frame-Options: DENY in netlify.toml headers (D-09) |
| T-03-03 | Tampering | MIME type interpretation | mitigate | X-Content-Type-Options: nosniff in netlify.toml headers (D-09) |
| T-03-04 | Information Disclosure | Referrer data leakage | mitigate | Referrer-Policy: strict-origin-when-cross-origin in headers (D-09) |
| T-03-05 | Tampering | XSS via content injection | mitigate | Content-Security-Policy in headers; MkDocs sanitizes markdown input (D-09) |
| T-03-06 | Tampering | Supply chain (GitHub Actions) | accept | actions/checkout@v6 and setup-python@v6 are verified releases; no custom Actions used; public docs site, low-value target |
</threat_model>
