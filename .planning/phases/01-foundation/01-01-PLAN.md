<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| pip/PyPI -> requirements.txt | External packages enter the build via pip install |
| mkdocs.yml -> MkDocs engine | Configuration drives the build; malformed YAML could cause silent failures |
| MkDocs build -> site/ | Build output is served to users; misconfiguration could leak assets |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-1-01 | Tampering | requirements.txt dependency pins | mitigate | Pin all dependencies with upper bounds: mkdocs>=1.6,<2.0, mkdocs-material>=9.7,<10.0, mkdocs-static-i18n==1.3.1, pymdown-extensions>=10.21,<11.0. Comments reference PIT-01 and PIT-02. |
| T-1-02 | Tampering | mkdocs.yml configuration | mitigate | Use `mkdocs build --strict` in verification (Plan 03) to catch malformed config. No dynamic YAML loading. |
| T-1-03 | Information Disclosure | offline/privacy plugin config | mitigate | Configure privacy plugin with `enabled: !ENV [OFFLINE, false]` so it activates in offline builds to localize external assets (D-13). Prevents CDN dependency leak in air-gapped environments. |
</threat_model>
