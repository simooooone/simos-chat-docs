# Phase 1 Foundation -- UI Design Contract

> Visual and interaction contract for MkDocs Material documentation portal. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | MkDocs Material 9.7.x (theme configuration) |
| Preset | Material default (no shadcn -- not a React project) |
| Component library | Material for MkDocs built-in components (admonitions, tabs, code blocks, search, nav) |
| Icon library | Material Icons (via `material/` prefix in mkdocs.yml) |
| Font | Roboto (text), Roboto Mono (code) -- Material defaults |

> This is a static documentation portal built with MkDocs Material, not a React/web app. There is no shadcn, no Tailwind, no component registry. The "design system" is the Material theme configuration in `mkdocs.yml` plus content conventions for stubs. The shadcn initialization gate is not applicable.

### Visual hierarchy

Primary focal point: documentation content area. Navigation sidebar is the secondary visual anchor.

Visual hierarchy: H1 page title (32px bold) > H2 section headings (24px bold) > body text (16px regular) > italic notes (16px regular italic).

---

## Spacing Scale

MkDocs Material manages its own layout spacing internally. The project uses default Material spacing with no custom CSS overrides (per D-06: default Material theme). The following tokens document Material's effective spacing values for reference by content authors and any future customization.

Declared values (multiples of 4px where applicable):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, list item spacing |
| sm | 8px | Compact element spacing, button padding |
| md | 16px | Default element spacing, paragraph gaps |
| lg | 24px | Section padding, card margins |
| xl | 32px | Layout gaps, sidebar padding |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing, hero section bottom |

Exceptions: Material's built-in touch targets for mobile (44px) are outside the 4px scale but are handled by the theme automatically.

### Content spacing conventions (for stub authors)

- Markdown heading followed by a blank line before body text
- One-line description paragraph immediately after heading
- Blank line before "Content coming soon" note
- Admonition blocks: standard Material spacing (no manual padding)

## Typography

Material theme defaults apply. No custom font configuration in mkdocs.yml (D-06: default Material theme). The `privacy` plugin (offline build) will localize Google Fonts, so Roboto loads from local assets in offline mode.

| Role | Size | Weight | Line Height | Source |
|------|------|--------|-------------|--------|
| Body | 16px | Regular (400) | 1.5 | Material default (`--md-text-font: Roboto`) |
| Code | 13.6px | Regular (400) | 1.6 | Material default (`--md-code-font: Roboto Mono`) |
| Heading (H2) | 24px | Bold (700) | 1.2 | Material default |
| Display (H1) | 32px | Bold (700) | 1.2 | Material default |

### Stub content typography conventions

- H1: Page title (one per stub)
- Body paragraph: One-line description
- Italicized note: `*Content coming soon.*` (English) / `*Contenuto in arrivo.*` (Italian)

## Color

Per D-04 (dark mode default) and D-06 (default Material theme, no custom brand colors). The palette uses Material's built-in `indigo` primary and `indigo` accent for both light and dark schemes.

### Dark mode (default, listed first per D-04)

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#202124` (slate background) | Page background, content area surfaces |
| Secondary (30%) | `#2e303e` (slate surface) | Sidebar, header bar, card backgrounds |
| Accent (10%) | `#536dfe` (indigo A200 on dark) | Active nav links, hovered links, search highlight, code copy button, inline code background |

Accent reserved for: active navigation tab, hovered links, search result highlights, code copy button, inline code background, interactive toggles.

### Light mode (toggle available)

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#ffffff` (default background) | Page background, content area surfaces |
| Secondary (30%) | `#f5f5f5` (default surface) | Sidebar, card backgrounds, code blocks |
| Accent (10%) | `#3f51b5` (indigo 500 on light) | Active nav links, hovered links, search highlight, code copy button, inline code background |

Accent reserved for: same elements as dark mode (active nav tab, hovered links, search result highlights, code copy button, inline code background, interactive toggles).

| Destructive | `#ff1744` (red A400) | Not used in Phase 1 (no destructive actions in static docs) |

### Theme palette configuration (mkdocs.yml)

```yaml
theme:
  palette:
    # Dark mode -- listed first = default (D-04)
    - scheme: slate
      primary: indigo
      accent: indigo
      toggle:
        icon: material/brightness-7
        name: Switch to light mode
    # Light mode
    - scheme: default
      primary: indigo
      accent: indigo
      toggle:
        icon: material/brightness-4
        name: Switch to dark mode
```

Source: CONTEXT.md D-04, D-06; RESEARCH.md Pattern 3.

## Navigation Structure

Per D-01 (feature-area-based organization), D-02 (8 top-level sections, 2-3 levels deep).

### Top-level navigation (8 tabs)

1. **Setup** -- installation, configuration
2. **Chat** -- getting started, features
3. **Documents & RAG** -- uploading, querying
4. **Workspaces & Projects** -- workspaces, projects
5. **Administration** -- RBAC, settings
6. **API** -- endpoints, authentication
7. **Deployment** -- Docker, air-gap setup
8. **Security** -- overview, compliance

### Navigation features (mkdocs.yml `theme.features`)

| Feature | Enabled | Rationale |
|---------|---------|-----------|
| `navigation.tabs` | Yes | D-02: Top-level sections as tabs |
| `navigation.sections` | Yes | D-02: Grouped sidebar sections |
| `navigation.indexes` | Yes | Section index pages |
| `navigation.top` | Yes | Back-to-top button |
| `navigation.trail` | Yes | Breadcrumbs |
| `navigation.footer` | Yes | D-12: Footer prev/next links |
| `search.suggest` | Yes | Search autocomplete |
| `search.highlight` | Yes | Highlight search results |
| `content.code.copy` | Yes | FDN-04: Code copy button |
| `content.code.annotate` | Yes | Code annotations support |
| `content.tooltips` | Yes | Hover tooltips for abbreviations |
| `navigation.instant` | **NO** | D-14: Excluded (breaks file:// protocol) |

### Other navigation

- **TOC sidebar** (right side): Enabled per D-11, configured via `toc` markdown extension with `permalink: true` and `toc_depth: 3`
- **Language switcher**: Configured via i18n plugin `reconfigure_material: true` and `extra.alternate`
- **Print-to-PDF**: Enabled per D-09 (built into Material, accessible via browser print dialog)

## Content Features (FDN-04)

These are not "UI components" in the React sense but are Material theme rendering features that must be configured and verified.

| Feature | mkdocs.yml Extension | Purpose |
|---------|---------------------|---------|
| Syntax highlighting | `pymdownx.highlight` + `pymdownx.inlinehilite` | Code blocks with line numbers and language detection |
| Admonitions | `admonition` + `pymdownx.details` | Note, warning, tip callout boxes |
| Content tabs | `pymdownx.tabbed` (alternate_style: true) | Tabbed code examples and content |
| Code copy button | `content.code.copy` (theme feature) | Copy button on code blocks |
| Superfences | `pymdownx.superfences` | Nested code blocks, Mermaid diagrams |
| Tables | `tables` | Markdown table rendering |
