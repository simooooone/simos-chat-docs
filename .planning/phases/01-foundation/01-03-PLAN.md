**Step 1: Install dependencies from requirements.txt**

Run the following command to install all pinned dependencies:

```bash
cd /home/simos/progs/Elia/simos-chat-docs
pip install -r requirements.txt
```

If this fails, requirements.txt or the package versions are wrong -- do NOT proceed, diagnose and fix.

**Step 2: Verify MkDocs version is 1.6.x (NOT 2.x)**

```bash
mkdocs --version
```

The output must contain "1.6" (e.g., "mkdocs, version 1.6.1"). If it shows "2." or any version >=2.0, STOP -- this means PIT-01 has occurred (MkDocs 2.0 breaks Material plugin system). The requirements.txt upper-bound pin has failed. Diagnose and fix before proceeding.

**Step 3: Run mkdocs build --strict**

```bash
cd /home/simos/progs/Elia/simos-chat-docs
mkdocs build --strict
```

This command validates the entire configuration:
- mkdocs.yml syntax and all plugin configurations
- All nav references point to existing files
- All markdown extensions load correctly
- No warnings or errors

If `--strict` fails, read the error output carefully. Common failures:
- "Page ... not found" -- a nav entry references a file that does not exist (check docs/ directory)
- "Plugin ... not found" -- a plugin name in mkdocs.yml does not match an installed package (check requirements.txt)
- "Config error" -- YAML syntax error or invalid plugin configuration (check mkdocs.yml)

Diagnose and fix any errors before proceeding. Do NOT remove `--strict` -- it is the primary validation tool.

**Step 4: Verify build output structure**

After a successful build, verify the site/ directory structure:

```bash
ls site/
```

Expected output should include directories for both languages (en/ and it/ subdirectories or similar structure generated by the i18n plugin).

**Step 5: Run automated constraint checks**

Run these grep-based checks to confirm locked decisions are enforced in the configuration:

```bash
# D-14: navigation.instant must NOT be in features
grep "navigation.instant" mkdocs.yml && echo "FAIL" || echo "OK: no navigation.instant"

# D-13: !ENV [OFFLINE, false] must be used
grep "OFFLINE" mkdocs.yml

# D-07: Plugin load order must be i18n -> search -> offline -> privacy
grep -n "i18n:\|search:\|offline:\|privacy:" mkdocs.yml

# FDN-02: MkDocs version check
mkdocs --version | grep "1.6"
```
